1. Java基础知识

1.1 基础概念

API：Application Programming Interface，即应用程序编程接口；
Java SE：Java Standard Edition，即Java标准版本，包含标准的JVM和标准的库；
Java EE：Java Enterprise Edition，即Java 企业版本，在Java SE的基础上加入大量的API和库，方便进行开发Web应用、数据库等；
注意：Java EE使用的JVM（Java虚拟机）和Java SE完全相同
Java ME：Java Micro Edition，即Java微版本，是一个针对嵌入式设备的版本，是Java SE的阉割版本，但是现在很少使用；
注意：Java SE的标准库在Java ME上无法使用，因为Java ME的JVM（Java虚拟机）也是阉割的版本

JDK：Java SE Development Kit，即Java标准版（Java SE）开发包；
Java EE SDK：Java EE Software Development Kit，即Java EE 软件开发包；
注意：Java EE SDK已经包含相应版本最新的JDK，所以安装Java EE SDK后就无需再安装JDK
JVM：Java Virtual Machine，即Java虚拟机；
JRE：Java Runtime Environment，即Java运行环境；

1.2 基本数据类型（Primeitive Type）

  • 整数型：byte、short、int、long
  • 浮点型：float、double
  • 字符型：char
  • 布尔型：false、true

基本数据类型还可以整体分为两类：数值类型、布尔型。
数值类型：整数型、浮点型、字符型，所有的数值型之间可以进行类型转换，包括自动类型转换和强制类型转换。
注意：int型是最常用的整数类型，在通常情况下，直接给出一个整数默认就是int类型（很重要，请记住）。
注意：char代表字符型，实际上字符型也是一种整数类型，相当于无符号整数类型。

计算机最小的内存单元就是字节（byte），一个字节就是8位二进制数，即8bit。它的二进制范围：0000000011111111，十进制范围：0255（即共2的7 次方=256个数，从0255共256个数），十六进制范围：00ff。
一个字节（byte）是1byte，1024byte=1K，1024K=1M，1024M=1G，1024G=1T。
注意1：对于整数型，最高位的bit表示符号位（0代表正数，1代表负数），各个整数型能表示最大的范围如下：
byte：-128 ~ 127（即-2^72^7-1），在内存里占8位（8bit）；
short：-32768 ~ 32767（即-2^152^15-1），在内存里占16位（16bit）；
int：-2147483648 ~ 2147483647（即-2^312^31-1），在内存里占32位（32bit）；
long：-9223372036854775808 ~ 9223372036854775807（即-2^632^63-1），在内存里占64位（64bit）；

注意2：Java语言对布尔类型的存储并没有做规定，因为理论上存储布尔类型只需要1bit（即用1个二进制位表示0或1），但是由于大部分计算机在分配内存时允许分配最小内存单元是字节（8位），所以1bit大部分实际占用8位。
注意3：所有数字在计算机底层都是以二进制形式存在的，正数是以原码的方式储存（可以直接转成十进制），负数是以补码的方式储存（补码不可以直接转成十进制，需要先将补码转成原码，再用原码转成十进制）。
负数转换计算原码的方法：计算机存储的负数补码——>补码减1得到反码——>反码——>符号位不变，其他按位取反——>原码

1.3 引用数据类型（Reference Type）

Java语言支持的类型分为两类：基本数据类型（Primitive Type）、引用数据类型（Reference Type）。
引用类型包括类、接口、数组类型，还有一种特殊的null类型。所谓引用类型就是对一个对象的引用，对象引用包括实例和数组两种。可以理解成C++中的指针概念，但是Java没有指针的说法。
null可以直接赋值给任何引用类型的变量，用以表示这个引用变量中保存的地址为空，即没有指向任何有效的对象。
注意：String类型的直接量不能赋给其他类型的变量，null类型的直接量可以直接赋给任何引用类型的变量。boolean类型的直接量（true、false）只能赋给boolean类型的变量，不能赋给其他类型的变量。

1.4 数据类型转换

Java所有的数值类型都可以相互转换，如果系统支持将某种基本类型直接赋给另一种类型的变量，则这种方式称为自动类型转换。如果不支持自动类型转换，那么就需要强制类型转换。
数据类型转换分为两种：自动类型转换、强制类型转换。

1.4.1 自动类型转换

1.4.2 强制类型转换

图3.10从左到右是自动类型转换，从右到左就是强制类型转换。强制类型转换的语法格式：（targetType）value。当进行强制类型转换时，是将数据大的类型向数据小的类型转换，在此过程，有可能会丢失数据，即造成溢出。

/*强制类型转换*/
int iValue = 233;
byte bValue = (byte)iValue; 
System.out.println(iValue); 
System.out.println(bValue); //输出bValue=-23，而原本iValue=233，即造成数据丢失，溢出。


从上图可以看出，强制类型转换是截取部分bit位保留，丢失其余bit位，这样很容易引起数据丢失。

1.4.3 表达式类型的自动提升

当一个算术表达式中包含多个基本类型的时候，整个算术表达式的数据类型会自动提升至最高类型。如一个算术表达式有byte、short、char这几种数据类型，Java会将byte——>int，short——>int，char——>int，再进行计算，计算后的结果为int型。如果算术表达式中含有long、float、double型，那么将会自动提升至long、float、double型。基本数据类型的自动转换规则参照图3.10。
具体类型自动提升规则如下：
1.所有的byte型、short型和char型的值将被提升到int型；
2.如果有一个操作数是long型，计算结果就是long型；如果有一个操作数是float型，计算结果就是float型；如果有一个操作数是double型，计算结果就是double型。

/*错误的写法*/
byte b1 = 10;
byte b2 = 5;
b1 = b1 + b2; //此时会报错，因为b1、b2都是byte类型，进行运算时，会将b1、b2提升至int型计算，即b1+b2结果是int型。等式右边为int型，等式左边为char型，两边数据类型不相等，不可以直接赋值。
System.out.println(b1);

/*正确的写法*/
byte b1 = 10;
byte b2 = 5;
b1 = (byte) (b1 + b2);// (b1 + b2)为int型，需要强制转换降级为byte型
System.out.println(b1);


/*错误的写法*/
short sValue = 5;
sValue = sValue-2;//sValue为short型，而2为int型，该算术表达式会自动提升为int型。等式右边为int型，等式左边为short型，两边数据类型不相等，不可以直接赋值。表数大的类型转换成表数小的类型，需要强制转换。

/*正确的写法*/
short sValue = 5;
sValue = (short) (sValue-2);// (sValue-2)为int型，需要强制转换降级short型

1.5 理解JVM(Java虚拟机)

当使用Java编译器（即Javac）编译Java源代码时，生成的是字节码，此时的字节码与平台无关（Windows、Linux等平台无法识别字节码，只能识别机器码），这些字节码只面向JVM。不同平台的JVM是不同的，但是他们都提供了相同的接口。通俗理解：JVM向上面对Java字节码提供相同的接口，向下面对不同的平台提供不同的接口，这样就可以实现：一套Java代码到处运行，而不必局限于平台。

2. 理解面向对象

2.1 面向对象特性

  • Java是面向对象编程，其有三种特性：封装（Encapsulation）、继承（Inheritance）、多态（Polymorphism）。Java语完全以对象为中心，Java程序的最小单位是类，整个Java程序由一个一个的类组成。
    封装：指的是将对象的实现细节隐藏起来，然后通过一些公用方法来暴露该象。
    继承：面向对象实现软件复用重要手段，当子类继承父类后，子类作为一种特殊的父类，将直接获得父类的属性和方法。注意：Java只允许单继承（只能继承一个父类），不允许多继承（可以继承多个父类）。
    多态：子类对象可以直接赋给父类变量，但运行时依然表现出子类的行为特征，这意味着同一个类型的对象在执行同一个方法时，可能表现出多种行为特征。
  • 一个Java项目只有一个主入口，即主函数Public static void main（String[] str）{},其余的类直接或者间接被主函数调用。一个类里面一般包含成员变量和方法，逻辑如下：
    $$成员变量（Field）+方法（Method）=类（Class）$$

    Java程序的组成单位就是类，不管多大的Java应用程序，都是由一个个类组成的。

2.2 面向过程和面向对象的区别

  • 面向过程和面向对象的区别：

面向过程：在面向过程的程序世界里，一切以函数为中心，函数最大，因此这件事情会用如下语句来表达：吃（猪八戒,西瓜）;
面向对象：在面向对象的程序世界里，一切以对象为中心，对象最大，因此这件事情会用如下语句来表达：猪八戒.吃（西瓜）。
Java语言是面向对象语言，所以从逻辑上来说更加符合人的正常思维。

  • 对象的抽象化就是类，类的实例化就是对象。
